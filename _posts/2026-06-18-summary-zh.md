## [字节跳动豆包 Agent 沙盒存在 S3 访问隔离漏洞](https://linux.do/t/topic/2427590) ⭐️ 9.0/10

安全研究人员发现，字节跳动的豆包 agent 沙盒未设置访问隔离，用户只需请求根目录即可读写其他用户的 S3 文件系统。 该漏洞暴露了所有豆包用户的敏感数据，可能导致数据泄露，损害用户对字节跳动 AI agent 服务的信任。 该漏洞在“办公任务”模式下通过要求 agent 检查“/”目录触发；截图证实了可以访问其他用户的文件。

rss · LINUX.DO 前沿快讯 · 6月18日 04:32

**背景**: Agent 沙盒是为 AI agent 安全执行任务提供的隔离运行环境。S3（Amazon Simple Storage Service）是一种云对象存储服务，正确的访问隔离对于防止跨租户数据泄露至关重要。该漏洞表明沙盒未能强制执行按用户或按租户的隔离。

## [本地 Qwen：互补工具，非劣质替代品](https://blog.alexellis.io/local-ai-is-not-opus/) ⭐️ 8.0/10

Alex Ellis 的博客文章认为，像 Qwen 这样的本地 AI 模型与 Claude Opus 等前沿模型是互补工具，而非劣质替代品，并且它们需要不同的提示技巧。 这一观点挑战了普遍认为本地模型只是前沿模型劣质版本的看法，强调了它们在离线运行和隐私保护等特定场景下的独特价值。 文章指出，本地模型在特定任务上可能更快，并提供更好的数据隐私，但它们的提示策略不同——通常更明确的提示比有时用于 Claude 的间接措辞效果更好。

hackernews · alphabettsy · 6月18日 03:04 · [社区讨论](https://news.ycombinator.com/item?id=48580209)

**背景**: 本地大语言模型（LLMs）在用户自己的设备上运行，无需联网，提供隐私保护和离线访问。像 Claude Opus 这样的前沿模型是来自 Anthropic 等公司的尖端云端专用系统。Qwen 是阿里巴巴开发的 LLM 家族，可在本地运行。

## [RFC 10008 定义新的 HTTP QUERY 方法](https://www.rfc-editor.org/info/rfc10008/) ⭐️ 7.0/10

RFC 10008 引入了 HTTP QUERY 方法，允许安全且幂等的请求携带请求体，填补了 GET 和 POST 之间的空白。 该新方法通过提供一种无副作用的复杂查询标准方式，可简化 API 设计，并且因为 QUERY 是幂等的且可缓存，可能改进缓存策略。 QUERY 方法安全且幂等，即重复相同的请求产生相同效果。缓存键包含请求体，这引发了缓存效率方面的考量。

hackernews · schappim · 6月17日 10:51 · [社区讨论](https://news.ycombinator.com/item?id=48568502)

**背景**: HTTP GET 安全且幂等，但不支持请求体；POST 支持请求体，但非幂等。这迫使开发者对复杂查询使用 POST 或将参数编码到 URL 中，导致缓存和安全性方面的限制。QUERY 方法通过允许请求体同时保持幂等性来解决此问题。
